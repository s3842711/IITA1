Project Idea

Overview

My idea for a project would be to create a game using Unity. It would be a strategy game where the player plays from the perspective of the typical enemy. The premise would be dungeon monsters fending off adventurers. The gameplay would include turn-based combat with combat being decided by the stats and positioning of each unit. The game will have levels or scenarios with the defeat condition being losing all your units or the enemy reaching the entrance for the next floor.

Motivation

I found this project to be interesting as most games would have the player play as a “good” protagonist or party. I also decided that to align with my interests in making a turn-based strategy game. With strategy games I've played previously generally enemies cannot interact in the same way with objects or items that the player can as often mechanics are made in the player's favour. I think it would be interesting if the player and enemy's roles were a bit more blurred with the enemies being able to pick up and equip weapons as well.

Description

This game will be a strategy game spilt into different levels. Each level the player will take control of units to prevent the enemy from reaching the door which would result in a level fail. The player will start off with their turn moving and attacking with their units, once a unit has acted an enemy will act and will continue alternating between player and enemy action. Combat will be dictated by the unit stats, equipment and positioning. Units would have stats relating to their offense and defence to determine their damage and durability. Equipment would augment their offense giving bonuses like range or effectivity and positioning would give damage bonuses if they were flanking or behind the enemy. Equipment will be on the floor or in chests for either the player or enemy to pick up so players would need to compete for such equipment if it is a viable decision for the player as the equipment would be dropped upon death.

For quality of life a turn order will be implemented to show in what order the enemy will move. This will help the player know which enemy will act so they can plan and adapt their strategy around that. This can help the player determine whether a unit should be moved sooner or later or when specific areas should be avoided or moved to. A range indicator will also be implemented for visual clarity on where it is dangerous to move their units. The range indicator will reduce the times a player will need to check an enemy's range as this may become an annoyance when there are multiple enemy's ranges to keep track of at a time.

The AI for the enemy shouldn't just be following a set path but be able to make dynamic choices depending on what is nearby. They should be able to make smart decisions on what they should do on their turn and what and where they should attack from.

As with most games this will include a tutorial to help players familiarise themselves with the game with any new mechanics being explained at the beginning of the level, so players are not lost when encountering something for the first time. The player may opt to close the tutorial early as to not have to read through it over and over when attempting levels multiple times or if they wish to learn about new mechanics as they play.

I foresee learning to make the logic for the AI to be an obstacle. I don't want the AI to make bad decisions and end up just running at the player mindlessly. I would need to figure out how to have it prioritise different actions be it for survivability or strategy. Another obstacle that may occur is getting started using Unity. Depending on how easy I find it to learn Unity it may take a long time before I am able to properly program things to work as they should. It may result in proper work not being done when they should and slow down the overall progress of development.

Tools and Technologies

To make this game, Unity should be used as it is an easy to learn engine for beginners with many tools available with it. There are tools such as Bolt which is a free visual scripting tool. There are various tutorials on how to use these tools with much of learning required for these tools being in learning how to perform visual scripting. There is also the option to code in C# which will take much longer to learn as it is much more complex in addition to needing to learn the syntax and semantics. Using Unity with Bolt should be used for this project to make it feasible to complete.

Skills Required

As this will be a single player strategy game, I will need to learn how to program the logic behind an AI. The AI should be able to make smart decisions with its own logic and not just follow set actions each turn. I would also need to learn how to program the logic for how each object interacts. How the player units interact with the enemy units which would include stats and the flank and rear mechanics would be an example of such logic. I would also be required to learn how to use state and flow graphs as this is how Bolt's visual scripting is used.

Outcome

By the end of this project a fun and challenging game will have been created. I will become experienced with some aspects of Unity which may help in future game development projects. In using Unity, I will grow accustomed to flow graphs and state graphs. I will also be able to learn how to program logic for events in addition to AI. As this is a group project it will further help me develop my communication skills as we may be able to bounce ideas between ourselves.
